# Space

We have a single scale of spacing values for all kinds of whitespace.

## Spacious

Since our typography is fluid, spacing is as well.
Medium space matches the font size of regular body text.
There are two smaller and three larger options available.
The two biggest sizes are mainly used for the grid; they grow faster in wider windows.

### Custom pairs

There are situations in which whitespace works best in one size in small windows, and another in larger ones.
One example is whitespace around a grid, which is very large on desktop screens, but must be limited on mobile phones.
Four combinations are available for this.

## How to use

- We use medium white space between the same components.
- Grids have `--ams-space-lg-xl` above and below them, and `--ams-space-lg-2xl` at its sides.

## Pixels and more

We use pixels in our documentation on sizes to convey a more accurate mental image.
In code, sizes are defined in relative units (like `rem`, `em`, `ex`, `vw`, `vh`, and `ch`) to make components more flexible and accessible.
MDN offers a good overview on these [CSS values and units](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_Values_and_Units).
The responsive values above are specified using the [clamp function](https://developer.mozilla.org/en-US/docs/Web/CSS/clamp).
