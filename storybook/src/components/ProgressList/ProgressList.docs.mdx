## Subcomponents

### Step

Each step in the list has a heading, a status, and optionally collapsible content.

<Canvas of={ProgressListStepStories.Step} />

<Controls of={ProgressListStepStories.Step} />

## Usage guidelines

### When to use

Assign a status to each step: _completed_ or _current_.
Steps without a status are considered _upcoming_.

Maintain a correct chronological order.
Only one step may be _current_; preceding steps must be _completed_.

Decide whether the user should be able to collapse and expand steps.

Progress List must only be used on a white background.

### How to use

Choose the correct level for the headings of the steps.

## Examples

### Controlled

Use `collapsed` and `onToggle` on each step to let a parent component drive the expand/collapse state.
The example below keeps only one step open at a time.

<Canvas of={ProgressListStories.Controlled} />

### Collapsible

Set `collapsible` to `true` to let users expand and collapse the content of each step.
Current and upcoming steps then start expanded.
Completed ones are initially collapsed.
Use the `defaultCollapsed` prop on a step to override the default behaviour.
The `onToggle` callback emits the expanded state whenever it changes.

<Canvas of={ProgressListStories.Collapsible} />

### With substeps

A step may contain a list of substeps.
They are visually subordinate to their parent step: their indicators are smaller, and they do not display a heading.

Don’t forget to set the `hasSubsteps` prop on the parent step.
