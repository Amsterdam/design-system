## Usage guidelines

### When to use

Use an Image Slider for a series of images that belong together.

### When not to use

Assume that some or many users will not navigate between the slides and only see the first image.
Display all images separately if you want each of them to receive attention.

### How to use

Provide at least 2 images and at most 5.
Feature the most essential image first.

A full-width Image Slider should run from one edge of the [Page](/docs/components-containers-page--docs) container to the other; position it next to a Grid, not in a Cell spanning all columns.
Smaller sliders are fine in a Grid Cell.

## Examples

### Responsive images

Provide a set of smaller and larger instances for every image (through `srcSet`, see [MDN](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/img#srcset)) to let the browser select the optimal source file.
A mobile device can often download a smaller file, saving the user’s bandwidth and time.
Don’t forget to still include the required `src` attribute.

<Canvas of={ImageSliderStories.ResponsiveImages} />

### With captions

Add a caption to any image to display a description below it, e.g. for context or attribution.
Keep captions short – they take up vertical space, especially on mobile.
Consistent caption lengths across images help avoid large layout shifts when sliding.

<Canvas of={ImageSliderStories.WithCaptions} />

## Design

The first or selected image shows at its maximum size.
Every image displays a thumbnail at about 20% of its width.
Users can use buttons, thumbnails, or swiping to navigate between the images.
The buttons are not displayed on a narrow touch device.
The images do not slide automatically.
