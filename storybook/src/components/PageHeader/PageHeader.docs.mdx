## Guidelines for elements

### Logo

- Use the correct logo: our main one, one for a sub-brand, or the English variation.
  See [Logo](/docs/components-media-logo--docs) documentation for details.
- The logo must have a link pointing to the homepage of the same website.
- For multistep forms hosted on a different domain, the logo links to its first step.

### Brand name

- The City’s main website or that of a sub-brand must not display a brand name, as the logo already includes a wordmark.
- All other websites must display the name of the brand or service next to the logo.
  This helps people understand they are on a different website and that its navigation is separate from the main one.
- For a multistep form hosted on a different domain, use the name of the form as the brand name.

### Inline menu

- The menu to the right can contain a maximum of 5 items, including the menu button.
- Labels should be short to help the menu fit on a single line whenever possible.
- A universal or learnable [icon](/docs/brand-assets-icons--docs) can be added to a menu item if it helps clarify it.
- Links move into the collapsible menu on narrow windows, unless they have the `fixed` prop.

### Menu button

- The ‘Menu’ button opens the collapsible menu.
  It does not appear when the Page Header has no child content.
- Use `noMenuButtonOnWideWindow` to hide the menu button on wide windows when there are only a few links.

### Collapsible menu

- This menu gets accessed through the menu button and offers much room for more links.
- Use `GridCellNarrowWindowOnly` to contain links from the inline menu that disappear in narrow windows.
- If the collapsible menu contains [Headings](https://designsystem.amsterdam/?path=/docs/components-text-heading--docs), give them level 2 and a size of ‘level-3’.
- Don’t add whitespace to the Grid in the collapsible menu — its container already provides this.

## Examples

### With moving links

Links can move from the inline menu to the collapsible one on narrow windows.
A `MenuLink` is hidden by default on narrow windows.
Use `GridCellNarrowWindowOnly` to show that same link in the collapsible menu on narrow windows.

If you do not want the `MenuLink` to be hidden, use the `fixed` prop.

In this example, ‘English’ moves to the collapsible menu, while ‘Zoeken’ does not.

<Canvas of={PageHeaderStories.WithMovingLinks} />

### Without menu button on wide windows

If you only have a few links, you may not always need a collapsible menu.
Use `noMenuButtonOnWideWindow` to hide the menu button on wide windows.
On narrow windows, the menu button will still be visible,
so do not forget to add the links to the collapsible menu.

<Canvas of={PageHeaderStories.WithoutMenuButtonOnWideWindow} />

### Without menu button

In some cases, a collapsible menu might not be necessary.
If the Page Header has no `children`, the menu button will not appear.
Remember to use the `fixed` prop if you want the inline menu links to stay in place.

<Canvas of={PageHeaderStories.WithoutMenuButton} />

### With custom logo link

The destination and accessible text of the logo link can be customized,
as well as the logo itself and its accessible name.

### With custom texts

The visible text of the menu button, its accessible texts and the accessible navigation description can be customized.

### With custom link component

Pass a custom link component to the `logoLinkComponent` prop to use your routing library for the logo link.
Here’s an example with the Link component of Next.js:

{/* prettier-ignore */}
```tsx
import { PageHeader } from "@amsterdam/design-system-react"
import NextLink from "next/link"

const LinkComponent = (props) => <NextLink {...props} />

return <PageHeader logoLinkComponent={LinkComponent} />
```

### With different branding

Websites for the City of Amsterdam **must** use one of our own logos and our menu icon.

Other organisations using this component can provide elements to display their own brand.
The image for the logo must be an svg, and its label should be ‘name of the organisation + logo’.
Set the appropriate values for the sizing tokens of [Logo](/docs/components-media-logo--docs).
A different icon for the menu button must be an svg as well.
