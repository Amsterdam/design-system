## Subcomponents

Provide the choices of the Select with Option subcomponents, and group related choices with a Group.
They render the native `<option>` and `<optgroup>` elements and accept their standard attributes only.

## Usage guidelines

### When to use

Use a Select when the user must pick one option from a list that is too long to display as Radios.

### When not to use

Avoid adding functionality to allow selecting multiple options.
Multi select is harder to use on desktop as they require the user to hold down the `Ctrl` or `Cmd` key while clicking on the options.
It is recommended to use checkboxes instead.

### How to use

Wrap a Select in a Field to associate it with a Label, description, and Error Message.

### In a Field

Use a Field to group a Select with a Label, description and / or an Error Message.

If you don’t need the description, remove its Paragraph and the `aria-describedby` from the Select.

Check [the Field docs](/docs/components-forms-field--docs) for more information on configuring it.

<Canvas of={SelectStories.InAField} />

### In a Field with validation

If the Select can become invalid, add an Error Message and its `id` to the `aria-describedby` attribute of the Select.

Check [the Field docs](/docs/components-forms-field--docs) for more information on configuring it.
