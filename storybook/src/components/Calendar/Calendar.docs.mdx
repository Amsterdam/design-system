## Usage guidelines

### When to use

Use a Calendar to display a month view for content organised by date, such as an event calendar.

### When not to use

Use a [Date Input](/docs/components-forms-date-input--docs) instead when the user needs to select a date in the context of a form.

### How to use

Display the month for the selected date, which may not be the current month.

Give it an accessible name with `accessibleName`.
The Calendar renders this name as a visually hidden heading and generates a unique `id` for it; pass `accessibleNameId` to set that `id` yourself.

#### Localisation

The Calendar defaults to Dutch.
Set `locale`, `accessibleName`, and the navigation button label props to use a different language; add `dir="rtl"` for right-to-left scripts such as Arabic.
The default value of `accessibleName` is 'Kalender'; update it so screen readers announce the landmark in the right language.
See the [Language developer guide](/docs/docs-developer-guide-language--docs) for the supported locales.

If your app uses Calendar in multiple places with the same language, a small wrapper component avoids repeating these props:

```tsx
const ArabicCalendar = (props: CalendarProps) => (
  <Calendar
    accessibleName="التقويم"
    dir="rtl"
    locale="ar-MA"
    nextMonthButtonLabel="الشهر التالي"
    nextYearButtonLabel="السنة التالية"
    previousMonthButtonLabel="الشهر السابق"
    previousYearButtonLabel="السنة السابقة"
    {...props}
  />
);
```

## Examples

### Dates without links

Each date can be a link, for example to a page that lists that day's events.
`linkTemplate` builds the target for a date; return `undefined` to render the date as plain text, or omit `linkTemplate` entirely to render a plain, non-interactive month.

## Features

### Routing library integration

Pass `linkComponent` to let a router supply its own link element, such as the one from Next.js.
Use it together with `linkTemplate`.
For more details, see the [routing libraries developer guide](/docs/docs-developer-guide-routing-libraries--docs).

This example for a `linkTemplate` function produces links like `?date=2026-12-04` for December 4, 2026:

{/* prettier-ignore */}
```ts
const formatDate = (date: Date) =>
  `${date.getFullYear()}-${String(date.getMonth() + 1).padStart(2, "0")}-${String(date.getDate()).padStart(2, "0")}`

const linkTemplate = (date: Date): string => `?date=${formatDate(date)}`
```

## Accessibility

The Calendar renders as a `nav` landmark.
Screen readers announce each link with its full date, including the weekday, so a day number is never read out of context.
