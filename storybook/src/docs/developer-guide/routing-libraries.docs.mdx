# Routing libraries

Our link components render a native anchor (`<a>`) by default.
To integrate a routing library ‚Äî such as Next.js or React Router ‚Äî pass its link component to the `linkComponent` prop.
The design system keeps its own styling, class names, and structure, and only swaps the underlying anchor for the component you provide.

Any anchor attributes you set on the design system component ‚Äî `href`, `target`, `data-*`, and so on ‚Äî are passed straight through to your `linkComponent`.
So in most cases you can pass the routing library‚Äôs link component directly, without writing a wrapper.

Provide a _stable_ `linkComponent`: a component defined at module level, or one created with `useCallback`.
Defining it inline (for example `linkComponent={(props) => <NextLink {...props} />}`) creates a new component type on every render, which forces React to unmount and remount the link ‚Äî losing focus and other state in the process.

Refs are forwarded only to the default anchor, not to a custom `linkComponent`.

Some examples for common routing libraries:

## Next.js

Modern Next.js link components render their own anchor and accept a `className` and children.
You can therefore pass `NextLink` directly as the `linkComponent` and give the `href` to the design system component, which passes it through:

{/* prettier-ignore */}
```tsx
import { Link } from "@amsterdam/design-system-react"
import NextLink from "next/link"

<Link href="/amsterdam" linkComponent={NextLink}>
  Link label
</Link>
```

`NextLink` is a stable, module-level reference, so the link is not remounted on every render.
This replaces the deprecated `legacyBehavior` and `passHref` props, which are no longer needed.

## React Router

React Router lets you hook into its routing with [the useLinkClickHandler hook](https://reactrouter.com/en/main/hooks/use-link-click-handler).
Wrap it in a hook that returns a `linkComponent`, like so:

{/* prettier-ignore */}
```jsx
import { Link } from "@amsterdam/design-system-react"
import { useCallback } from "react"
import { useHref, useLinkClickHandler } from "react-router-dom"

const useReactRouterLink = (to) => {
  const href = useHref(to)
  const handleClick = useLinkClickHandler(to)

  return useCallback(
    function ReactRouterLink({ onClick, ...props }) {
      return (
        <a
          {...props}
          href={href}
          onClick={(event) => {
            onClick?.(event)
            if (!event.defaultPrevented) handleClick(event)
          }}
        />
      )
    },
    [href, handleClick],
  )
}

const Example = () => {
  const linkComponent = useReactRouterLink("/amsterdam")

  return <Link linkComponent={linkComponent}>Link label</Link>
}
```

`useCallback` keeps the component reference stable across renders, so the link is not remounted.
Composing `onClick` preserves any handler passed to the design system component, and the `defaultPrevented` check lets that handler opt out of client-side navigation when needed.
