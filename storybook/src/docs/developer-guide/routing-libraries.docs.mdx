# Routing libraries

Our link components render a native anchor (`<a>`) by default.
To integrate a routing library such as Next.js or React Router, pass its link component.

## Next.js

Modern Next.js link components render their own anchor and accept a `className` and children.
You can pass `NextLink` directly as the `linkComponent` and give the `href` to the design system component, which passes it through:

{/* prettier-ignore */}
```tsx
import { Link } from "@amsterdam/design-system-react"
import NextLink from "next/link"

<Link href="/amsterdam" linkComponent={NextLink}>
  Link label
</Link>
```

`NextLink` is a stable, module-level reference, so the link is not remounted on every render.
This replaces the deprecated `legacyBehavior` and `passHref` props, which are no longer needed.

## React Router

For React Router, wrap [the useLinkClickHandler hook](https://reactrouter.com/en/main/hooks/use-link-click-handler) in a hook that returns a link component, like so:

{/* prettier-ignore */}
```jsx
import { Link } from "@amsterdam/design-system-react"
import { useCallback } from "react"
import { useHref, useLinkClickHandler } from "react-router-dom"

const useReactRouterLink = (to) => {
  const href = useHref(to)
  const handleClick = useLinkClickHandler(to)

  return useCallback(
    function ReactRouterLink({ onClick, ...props }) {
      return (
        <a
          {...props}
          href={href}
          onClick={(event) => {
            onClick?.(event)
            if (!event.defaultPrevented) handleClick(event)
          }}
        />
      )
    },
    [href, handleClick],
  )
}

const Example = () => {
  const linkComponent = useReactRouterLink("/amsterdam")

  return <Link linkComponent={linkComponent}>Link label</Link>
}
```

`useCallback` keeps the component reference stable across renders, so the link is not remounted.
Composing `onClick` preserves any handler passed to the design system component, and the `defaultPrevented` check lets that handler opt out of client-side navigation when needed.

## Good to know

The design system components keep their own styling, class names, and structure, and only swap the underlying anchor for the component you provide.
Any anchor attributes like `href`, `target`, and `data-*` are passed straight through to your `linkComponent`.
So in most cases you can pass the routing library’s link component directly, without writing a wrapper.

Provide a _stable_ `linkComponent`: one that is defined at module level, or one created with `useCallback`.
Defining it inline (for example `linkComponent={(props) => <NextLink {...props} />}`) would create a new component type on every render, which forces React to unmount and remount the link, losing focus and other state in the process.

Refs are forwarded only to intrinsic elements – the default `<a>`, or a string tag such as `linkComponent="a"` – not to a custom `linkComponent`, which may not accept one.
If you need a ref to a custom link component, set it on that component directly.
