# Space between

Components handle their own internal whitespace, but the space between them is yours to apply.

## How to use

- Find the component on top in the column headers, then the component below in the row headers.
  The cell where they meet shows the recommended [spacing size](/docs/brand-design-tokens-space--docs).
- Apply that size in Figma.
  Developers can then apply it using the corresponding [margin class name](/docs/utilities-css-margin--docs) on each component.

## Components that use their own Grid Cell

An **Alert**, **Breadcrumb**, **Breakout**, and **Image Slider** must each have their own Grid Cell.
Hence, the space around them will be `xl`, or `2xl` horizontally if they sit at the start and/or end of the Grid.

## Notes

- Our spacing is responsive and also different between Spacious and Compact Mode.
  It’s not necessary to consider the pixel values; just work with the t-shirt sizes.
- If the cell is empty, or a component doesn’t have a row in a table, this means the combination should not occur.
- For an **Accordion**, see **Heading** with the level you use for it.
- **… List** refers to Description List, Link List, Ordered List and Unordered List.

## Code example

{/* prettier-ignore */}
```tsx
<Grid.Cell>
  <Heading className="ams-mb-s" level={1}>
    Met korting van A naar B op de deelscooter of -bakfiets
  </Heading>
  <Paragraph className="ams-mb-xl">29 juli 2025</Paragraph>
  <Paragraph size="large">Woont of werkt u in Amsterdam? Dan maakt u tot en met oktober met korting gebruik van deelscooters en -bakfietsen. Zo kunt u de auto laten staan en de werkzaamheden in en rond de stad vermijden.</Paragraph>
</Grid.Cell>
```
