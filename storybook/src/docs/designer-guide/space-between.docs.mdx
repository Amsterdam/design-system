# Space between

Components handle their own internal whitespace, but the space between them is yours to apply.

## Design

- Spacing is responsive and differs per [mode](/docs/docs-developer-guide-modes--docs).
  That’s why we use t-shirt sizes, not pixel values.
- In general, space between two equal or comparable components is medium.
- Space between containers is large.
  Space between grid columns is x-large.

## How to use the tables

- For every combination of two components, find the top one in the column headers and the one below in the row headers.
  The cell where they meet shows the recommended [spacing size](/docs/brand-design-tokens-space--docs) between them.
- Apply that size in Figma.
  Developers use the corresponding [margin class](/docs/utilities-css-margin--docs).
- An empty cell or a missing component means we advise against that combination.
- ‘… List’ refers to Description List, Link List, Ordered List, and Unordered List.

## Components that have their own Grid Cell

**Alert**, **Breadcrumb**, **Breakout**, **Image Slider**, and **Spotlight** must be in their own [Grid Cell](/docs/components-layout-grid--docs).
No extra spacing is needed.

## Code example

{/* prettier-ignore */}
```tsx
<Grid.Cell>
  <Heading className="ams-mb-s" level={1}>
    Met korting van A naar B op de deelscooter of -bakfiets
  </Heading>
  <Paragraph className="ams-mb-xl">29 juli 2025</Paragraph>
  <Paragraph size="large">Woont of werkt u in Amsterdam? Dan maakt u tot en met oktober met korting gebruik van deelscooters en -bakfietsen. Zo kunt u de auto laten staan en de werkzaamheden in en rond de stad vermijden.</Paragraph>
</Grid.Cell>
```
