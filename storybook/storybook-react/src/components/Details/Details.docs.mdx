## Stories

### With heading

_Details_ kan een _heading_ niveau meekrijgen zodat de titel in de inhoudsopgave van de pagina terecht komt.
Als je bijvoorbeeld wilt dat mensen die gebruikmaken van een schermlezer direct naar het _details_ component kunnen navigeren, kun je deze _property_ meegeven.
Gebruik hierbij een passend _heading_ niveau. Als je details component bijvoorbeeld onderdeel is van een sectie die met _heading_ niveau 2 begint, geef dan `headingLevel` 3 mee.

### Open by default

Als je wilt dat de inhoud van een _details_ component al getoond wordt als de gebruiker op de pagina komt, geef dan de `open` _property_ mee.

## Technische toelichting

Op het moment (28-6-2023) heeft het HTML element `details` wat toegankelijkheidsproblemen.
[Zo wordt een _heading_ in een `summary` niet goed begrepen door schermlezers](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/summary#summaries_as_headings) en
leest NVDA de ingeklapte en en uitgeklapte status niet goed voor in Chrome en Edge. Daarom hebben we er voor gekozen om het _native_ element niet te gebruiken.
Als deze problemen zijn opgelost, zouden we dit wel kunnen doen.
