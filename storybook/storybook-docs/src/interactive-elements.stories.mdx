# Interactieve elementen

Hiermee bedoelen we elementen waarmee de gebruiker kan interacteren, zoals links, knoppen en formuliervelden. Ze moeten duidelijk herkenbaar zijn en gemakkelijk te onderscheiden van niet-interactieve elementen. Je moet ze op alle gangbare manieren kunnen bedienen, zoals met een muis, toetsenbord en touchscreen.

## Tekst

Interactieve elementen moeten duidelijk zijn, gebruikers moeten kunnen voorspellen wat er gebeurt als je een element activeert. Een interactief element moet ook duidelijk zijn zonder dat je de zinnen er omheen gelezen hebt. Gebruik geen onduidelijke tekst, zoals "hier" of "lees meer".

### Relevante WCAG eisen:

- [WCAG 2.4.4](https://www.w3.org/TR/WCAG22/#link-purpose-in-context): het is voor gebruikers duidelijk wat het betekent om op een link te klikken
- [WCAG 2.4.6](https://www.w3.org/TR/WCAG22/#headings-and-labels): gebruik een label om interactieve elementen te beschrijven
- [WCAG 2.5.3](https://www.w3.org/TR/WCAG22/#label-in-name): de toegankelijke naam van een element moet in ieder geval de zichtbare tekst van het element bevatten

## Kleur

Blauw is de interactieve kleur, deze wordt bijna altijd gebruikt voor interactieve elementen. Bij hover wordt dit donkerblauw. Een uitzondering hierop zijn error states, die gebruiken rood en donkerrood voor hover.

Kleur mag niet de enige indicator van interactiviteit zijn, zodat mensen die moeite hebben met het waarnemen van verschillen in kleur toch kunnen zien dat een element interactief is.

### Relevante WCAG eisen:

- [WCAG 1.4.1](https://www.w3.org/TR/WCAG22/#use-of-color): kleur mag niet de enige manier zijn waarop je aangeeft dat iets interactief is
- [WCAG 1.4.3](https://www.w3.org/TR/WCAG21/#contrast-minimum): tekst heeft voldoende contrast met de achtergrond

## Formaat

[Interactieve elementen moeten minstens 24 bij 24 pixels groot zijn.](https://www.w3.org/WAI/WCAG22/Understanding/target-size-minimum.html)

## Focus indicator

Wij gebruiken grotendeels de standaard focus indicator van de webbrowsers. De enige aanpassing die we doen is de focus `outline` een `offset` geven van 2 pixels, zodat er ruimte zit tussen het element en de focus indicator. Hierdoor heeft de focus indicator ook voldoende contrast met donkere elementen.

Let op bij het inrichten van een pagina dat de interactieve elementen in een logische volgorde staan, meestal van boven naar beneden en van links naar rechts. Hierdoor krijg je een logische focus / tabbing order.

### Relevante WCAG eisen

- [WCAG 2.4.3](https://www.w3.org/TR/WCAG22/#focus-order): interactieve elementen hebben een logische focus / tabbing order
- [WCAG 2.4.7](https://www.w3.org/TR/WCAG22/#focus-visible): de focus indicator is zichtbaar
- [WCAG 2.4.11](https://www.w3.org/TR/WCAG22/#focus-not-obscured-minimum): de focus indicator wordt niet volledig verborgen door content
- [WCAG 2.4.12](https://www.w3.org/TR/WCAG22/#focus-not-obscured-enhanced): de focus indicator wordt niet deels verborgen door content
- [WCAG 2.4.13](https://www.w3.org/TR/WCAG22/#focus-appearance): de focus indicator is minimaal 2 pixels breed en heeft voldoende contrast met het element dat focus heeft

## Hover

Gebruik verschillende, niet overlappende stijlen voor hover en focus. Als toetsenbordgebruiker is het belangrijk dat je weet welk element je activeert. Als de hover-variant (bijna) hetzelfde is als de focus-variant, dan weet je niet zeker welk element je zult activeren.

We gebruiken de cursor hand `pointer` bij hover op alle interactieve elementen die geen eigen cursor stijl hebben (zoals tekstvakken). Oorspronkelijk was `pointer` alleen bedoeld voor links, maar wij vinden dat de associatie van `pointer` met interactiviteit zo sterk is dat we deze cursor vaker gebruiken.

Gebruik het `title` attribuut niet. Interactieve elementen met het `title` attribuut kunnen een tooltip tonen die over de tekst heen staat, waardoor het minder leesbaar is.

### Referenties

- [Documentatie van Utrecht over focus en hover](https://nl-design-system.github.io/utrecht/storybook/?path=/docs/css-link--docs)
- [Specificatie van cursor pointer.](https://www.w3.org/TR/css-ui-4/#valdef-cursor-pointer) Hierin staat dat je pointer moet gebruiken voor links, en mag gebruiken voor andere interactieve elementen

## Dubbelklikken

[Sommige gebruikers zijn gewend om te dubbelklikken op interactieve elementen.](https://design-system.service.gov.uk/components/button/#stop-users-from-accidentally-sending-information-more-than-once) Zorg ervoor dat dit niet tot problemen leidt, bijvoorbeeld door een loading state te gebruiken en / of op de server te controleren of data niet vaker is verzonden.

Om ervoor te zorgen dat je niet per ongeluk inzoomt op apparaten die double tap zoom ondersteunen, gebruiken we de CSS regel ` touch-action: manipulation` op alle interactieve elementen.
